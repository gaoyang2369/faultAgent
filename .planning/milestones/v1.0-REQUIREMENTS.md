# Requirements: AI Agent 后端模块化重组

**Defined:** 2026-03-26
**Core Value:** 模块清晰可替换 — 新用户 fork 后替换 tools/、prompts/、middleware.py、config.py 即可搭建自己的 Agent 服务

## v1 Requirements

Requirements for modular restructuring. Each maps to roadmap phases.

### Safety Net

- [x] **SAFE-01**: Characterization test 覆盖 SSE 流式响应（验证 start/token/tool_start/tool_end/complete 事件序列）
- [x] **SAFE-02**: Characterization test 覆盖工具调用（至少验证 get_time 和 sql_inter 工具能被正确触发和返回）
- [x] **SAFE-03**: Characterization test 覆盖历史 API（/ai/history/{type} 和 /api/todos/{thread_id} 端点返回正确格式）
- [x] **SAFE-04**: Smoke test 脚本验证 agent 启动成功并能响应简单查询

### Config & Utils

- [x] **CONF-01**: 创建 config.py 集中管理所有硬编码配置值（Ollama URL、embedding model、FAISS path、max_tokens、messages_to_keep、recursion_limit、DCMA db name、ML API URL）
- [x] **CONF-02**: 创建 utils.py 包含通用工具函数（sanitize_for_json、safe_json_dumps、parse_todos 系列）
- [x] **CONF-03**: 现有文件（app.py、tools.py、knowledge_base.py）中的硬编码值和通用函数改为从 config.py / utils.py 导入

### Tools Modularization

- [x] **TOOL-01**: 创建 tools/ 目录，按关注点分文件（data_tools、sql_tools、kb_tools、report_tools、utility_tools）
- [x] **TOOL-02**: 模块级 DB 连接（tools.py:32-48 的 SQLDatabase + SQLDatabaseToolkit）改为延迟初始化
- [x] **TOOL-03**: extract_data + fig_inter + python_inter 保持在同一文件 tools/data_tools.py，globals() 共享正常
- [x] **TOOL-04**: subagent/ 迁移到 tools/subagent/，fault_explanation_tool 从新位置导入
- [x] **TOOL-05**: tools/__init__.py 导出完整的 tools 列表，旧 tools.py 删除

### Prompts & Middleware

- [x] **PROM-01**: 创建 prompts/ 目录，system_prompt.py 包含 systemprompt，dynamic_prompt.py 包含 Context + @dynamic_prompt
- [x] **PROM-02**: 创建 middleware.py，中间件列表组装逻辑从 lifespan 中提取
- [x] **PROM-03**: 旧 prompt_template.py 删除

### Knowledge Base

- [x] **KBAS-01**: knowledge_base.py 的 Ollama URL、embedding model、FAISS path 从 config.py 读取
- [x] **KBAS-02**: 保留 8 秒超时保护机制
- [x] **KBAS-03**: rebuild_kb.py 适配新结构后可正常执行

### App Slim & Integration

- [x] **SLIM-01**: app.py 中不包含工具定义、提示词内容、通用工具函数
- [x] **SLIM-02**: app.py 行数不超过 300 行
- [x] **SLIM-03**: 所有现有 API 端点行为不变，前端无需任何修改
- [x] **SLIM-04**: PostgreSQL 会话状态持久化正常工作

## v2 Requirements

Deferred to future release.

### Developer Experience

- **DEVX-01**: 项目脚手架模板（cookiecutter/copier），一键创建新项目骨架
- **DEVX-02**: 启动时自动验证配置完整性
- **DEVX-03**: 结构化日志替换 print 语句

### Advanced Features

- **ADVF-01**: Tool 自动发现（扫描 tools/ 目录自动注册）
- **ADVF-02**: pydantic-settings 配置验证升级
- **ADVF-03**: 多 Agent 实例支持

## Out of Scope

| Feature | Reason |
|---------|--------|
| 前端 Vue 重构 | 本次只重构后端 Python，前端保持现状 |
| 数据库架构变更 | MySQL + PostgreSQL 统一不变 |
| 新业务功能 | 纯重构，不新增功能 |
| pip 包发布 | 不将项目发布为独立包 |
| Protocol/ABC 抽象 | 不做框架级抽象，保持简单 |
| agent_core/ + projects/ 分层 | 不做双层架构 |
