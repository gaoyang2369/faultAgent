# 长任务 / 流式 / 历史线程稳定性定向修复计划

日期：2026-04-08

## 目标

- 验证并修复三条主因：
  1. 知识库首次冷启动过重；
  2. 模型异步流式调用偶发中断或长时间无首事件；
  3. 历史列表与 thread 恢复存在次级稳定性问题。

## 执行顺序

1. 先做纯 SQL、纯知识库、纯模型流式、历史线程四组基线验证。
2. 仅在证据成立后进行最小必要修复：
   - 在线知识库查询不再承担首次建库；
   - SSE 在安全条件下回退到稳定的非流式调用；
   - 历史接口修正 `alist(None)`；
   - 前端 API 基地址按当前 hostname 对齐，降低 cookie / thread 失配。
3. 用自动化测试和真实环境验证修复结果。
