# Plan

## Goal

支持用户编辑已经发送的文本消息，并从该消息处重新生成回答，前端展示覆盖原本的后续回答。

## Scope

- 在聊天消息组件中为用户消息增加编辑、取消、保存入口。
- 在聊天流状态中提供编辑回滚和重新生成逻辑。
- 如需同步服务端上下文，补充轻量历史编辑接口，避免旧 checkpoint 继续污染重新生成。
- 增加覆盖关键行为的测试或运行现有相关测试。

## Notes

- 不读取 `.env` 内容。
- 不改变既有聊天发送和历史读取接口的语义。
- 先实现文本消息编辑，连续发送/排队策略后续再处理。
