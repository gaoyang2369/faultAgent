# Streaming Tool Render Bugfix 计划

## 问题现象
- 复杂问答时，助手正文中出现原始 SQL 语句或工具参数
- 工具执行结果在前端消息区显示为 `[object Object]`
- SSE 中途失败时，界面停留在中间态，最终回答缺失

## 本轮目标
- 后端抑制不应直接展示给用户的工具调用阶段内容
- 前端对对象型工具结果做可读摘要，而不是直接 `String(object)`
- 增加回归测试，防止问题再次出现
- 运行测试、构建和最小真实探针验证

## 执行步骤
1. 盘点后端流式 token 输出路径和前端工具事件展示路径
2. 实施最小侵入修复
3. 为工具结果摘要和流式过滤增加测试
4. 运行 pytest 与前端构建
5. 做一次真实后端流式探针验证

## 风险控制
- 不改 HTTP 接口契约
- 不改工具执行逻辑，只改展示与流式输出策略
- 优先避免误伤正常自然语言 token
