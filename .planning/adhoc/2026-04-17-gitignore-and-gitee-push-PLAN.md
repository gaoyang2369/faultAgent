# Git 忽略与 Gitee 推送准备计划（2026-04-17）

## 背景

当前仓库准备首次通过命令行推送到 Gitee。推送前需要先收口 `.gitignore`，避免把本地敏感配置、缓存、日志和临时调试产物一起提交。

## 目标

1. 审计当前 `.gitignore` / 前端子目录 `.gitignore`
2. 补充本轮本地产生但尚未忽略的目录与文件类型
3. 说明哪些内容已经被忽略，哪些内容仍应人工确认
4. 给出从本地仓库推送到 Gitee 的逐步命令行流程

## 执行步骤

1. 检查现有忽略规则与未跟踪文件分布
2. 追加最小必要的忽略项，如本地 npm 缓存、临时编译目录、SQLite 临时库、pytest 本地临时目录
3. 复核 `check-ignore` 结果
4. 输出可直接执行的 git 初始化、远端绑定、提交、推送步骤

## 风险

- `.gitignore` 只能阻止“后续新增/未跟踪文件”进入暂存区，若文件已被跟踪，需要额外 `git rm --cached`
- 首次推送前仍应人工检查 `git status` 和 `git diff --cached`，确保没有误加敏感文件
