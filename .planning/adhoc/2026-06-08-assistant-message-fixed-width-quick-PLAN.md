# 助手消息固定宽度 quick 计划

## 目标

修复助手回复在执行阶段随着状态、任务面板和最终内容变化而逐步变宽的问题，让助手消息从开始渲染时就保持稳定宽度。

## 范围

- 定位聊天消息容器和任务/摘要/最终回答面板的宽度来源。
- 优先通过前端样式约束 assistant 消息主容器宽度，不改 SSE 协议和消息数据结构。
- 保持用户消息继续按内容自适应。

## 验证

- 运行前端布局检查脚本。
- 运行前端生产构建或可用的轻量校验。
- 运行 `git diff --check`。
