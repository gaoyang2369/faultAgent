# 2026-04-28 PDF 手动归档 CTA 修复计划

## 背景
- 当前“上传并登记”会让用户误以为 PDF 已可被 Agent 使用。
- 实际上上传链路缺少显式“知识库归档”动作，用户无法主动完成入库闭环。
- 右侧仍保留“浏览器打印导出”，占用了核心 CTA 位置。

## 目标
1. 上传后默认只完成文件登记与轻量文本提取。
2. 新增显式“知识库归档”接口和前端按钮。
3. 归档成功后，上传 PDF 内容能被 `query_knowledge_base` 查询到。
4. 归档失败、需要 OCR、已归档等状态前后端一致。

## 执行步骤
1. 审计上传记录、文本提取、知识库构建与查询入口。
2. 后端拆分“提取”和“归档”两个动作。
3. 新增 `POST /admin/pdfs/{record_id}/ingest`。
4. 前端替换打印按钮为“知识库归档”，并增加 Agent 可读取状态展示。
5. 补定向测试、构建和闭环验证。
