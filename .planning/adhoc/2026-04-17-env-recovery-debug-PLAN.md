# 2026-04-17 `.env` 恢复排查计划

## 背景

- 用户怀疑此前代码整理时误删了项目根 `.env`。
- 目标是先确认删除轨迹，再在不暴露敏感内容的前提下，从历史或其他分支恢复。

## 执行步骤

1. 检查当前工作区 `.env` 是否存在、是否为空文件。
2. 审计 git 历史与分支中 `.env` 的新增 / 修改 / 删除轨迹。
3. 若确认存在可恢复版本，则直接从删除前最后一个版本恢复到工作区。
4. 用非内容型校验确认恢复结果，例如文件大小、行数与来源提交。

## 风险控制

- 不打印 `.env` 内容。
- 不把 `.env` 重新加入 git 跟踪。
- 仅恢复到工作区，避免影响当前分支历史。
