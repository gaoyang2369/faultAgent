# 文本输入重复回复调试计划

## 目标

定位游客状态下文字输入触发两次回复、语音输入只触发一次回复的原因，并补上轻量防重入保护。

## 范围

- 对比文字发送路径 `useChatStream.sendMessage` 与语音网关消息合并路径。
- 检查输入框 Enter、发送按钮、桌宠事件等入口是否共享发送逻辑。
- 在共享发送入口增加同步防重入锁，避免同一帧重复触发创建两条 SSE 流。

## 验证

- 运行前端生产构建。
- 检查本次改动范围，确认不读取 `.env` 内容、不改 HTTP API 契约。
