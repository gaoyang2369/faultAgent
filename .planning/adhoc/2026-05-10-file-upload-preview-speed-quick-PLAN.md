# 文档识别校对体验修复计划

## 目标

修复文档识别校对弹窗的两个体验问题：PDF 上传后保持固定窗口尺寸，并允许在原文件预览区域内用鼠标滚轮上下浏览；同时排查识别接口返回后前端结果显示偏慢的问题。

## 范围

- 调整 `FileUpload` / PDF 预览相关前端布局，保持上传前后面板高度稳定。
- 确保 PDF 原文件预览在固定容器中滚动，不撑高整个弹窗。
- 排查识别请求、轮询或渲染链路中的延迟点，并做轻量优化。
- 不改动既有 HTTP endpoint 契约。

## 验证

- 前端构建通过。
- 上传 PDF 后原文件区域高度稳定，滚轮可在预览区域内滚动查看。
- 识别完成后 Markdown 与文档预览能更快更新。
